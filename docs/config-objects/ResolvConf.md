# ResolvConf

## 用途

`ResolvConf` 由 `network.resolvconf` 插件使用，用来托管系统 resolver 文件
`/etc/resolv.conf`。

它只描述本机 resolver 文件本身，不负责 dnsmasq、Docker DNS、China Names、Unbound
等 DNS 服务配置。那些配置仍由各自 ConfigObject 管理。

这是单例配置，名称固定为 `system`。没有该对象时，DROS 不会接管
`/etc/resolv.conf`。

## 内置配置

DROS 没有内置 `ResolvConf` YAML。也就是说，如果用户配置目录里没有该对象，
`gw update` 不会写 `/etc/resolv.conf`。

代码中也没有默认 nameserver，因为 resolver 入口取决于实际部署：有些网关会指向本机
dnsmasq，有些会临时指向外部公共 DNS。

## 常见配置

```yaml
apiVersion: dros/v1alpha1
kind: ResolvConf
metadata:
  name: system
spec:
  nameservers:
    - 10.0.0.1
    - 223.5.5.5
  search:
    - lan
  options:
    - timeout:2
    - attempts:2
```

渲染结果：

```text
# Generated by DROS. Manual changes will be overwritten.
# Resource: ResolvConf/system
nameserver 10.0.0.1
nameserver 223.5.5.5
search lan
options timeout:2 attempts:2
```

## 字段

### `apiVersion`

当前建议写 `dros/v1alpha1`。如果写了该字段，当前必须为 `dros/v1alpha1`；省略时按当前版本处理。

默认值：无。

### `kind`

必须为 `ResolvConf`。

默认值：无。

### `metadata.name`

固定为 `system`。

默认值：如果省略，loader 视为 `default`，但 `network.resolvconf` 会拒绝非 `system`
名称。

### `metadata.disabled`

设置为 `true` 后，该对象会被忽略。注意这不会删除已有 `/etc/resolv.conf`。

默认值：`false`。

### `spec.nameservers`

nameserver 列表。每一项渲染为一行 `nameserver ...`。

默认值：`[]`。对象存在时必须至少配置一项。

### `spec.search`

搜索域列表。非空时渲染为一行 `search ...`。

默认值：`[]`。

### `spec.options`

resolver options 列表。非空时渲染为一行 `options ...`。

默认值：`[]`。

## CLI

```sh
gw update resolvconf
gw update resolvconf/system
gw config create resolvconf
```
